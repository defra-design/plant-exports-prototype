Accessibility statement for the 'Apply for a phytosanitary certificate' service

This service is part of the wider GOV.UK website. There’s a separate accessibility statement for the main GOV.UK website. This page only contains information about the phytosanitary certificate online service.
  1. Using this service

    This service is run by the Animal and Plant Health Agency (APHA). We want as many people as possible to be able to use this service. For example, that means you should be able to:

    • change colours, contrast levels and fonts
    • zoom in up to 300% without the text spilling off the screen
    • get from the start of the service to the end using just a keyboard
    • get from the start of the service to the end using speech recognition software
    • listen to the service using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

    We've also made the text in the service as simple as possible to understand.

    AbilityNet has advice on making your device easier to use if you have a disability.

  2. How accessible this service is

    Some people may find parts of this service difficult to use:

    • the text will not reflow in a single column when you change the size of the browser window
    • you cannot modify the line height or spacing of text
    • some of our online forms are difficult to navigate using just a keyboard
    • you cannot skip to the main content when using a screen reader
  3. What to do if you have difficulty using this service

  5. Enforcement procedure

    6. The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations'). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

  6. Contacting us by phone or visiting us in person

    7. We provide a text relay service for people who are D/deaf, hearing impaired or have a speech impediment.

    Our offices have audio induction loops, or if you contact us before your visit we can arrange a BSL interpreter.

    Contact APHA.

  7. Technical information about this website's accessibility

    8. The Department for Environment, Food and Rural Affairs (Defra) and APHA are committed to making their websites accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

    This service is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard. The non-accessible parts are listed below.

  8. Non-accessible content

    9. Some content on this website is non-accessible

    • some radio buttons are missing form labels - this fails WCAG 2.1 AA 2.4.6 Section 508 (2017) AA 2.4.6
    • there is a broken link on one form screen - the link does not have a target which fails WCAG 2.1 A 2.4.1 Section 508 (2017) A 2.4.4
  9. Content that’s not within the scope of the accessibility regulations

    10. Some accessibility problems fall outside the scope of the accessibility regulations

    • there is an issue with tab order on some screens which may affect how users navigate from one form field to another if they are using the tab key
    • users of Narrator screen reader x3 may find it difficult to use a keyboard to make a selection from one of the drop down lists
    • users of all types of screen reader users may find it difficult to make a selection from two of the drop down lists
    • there is an issue with page zoom on the dashboard screen which means some column titles are not visible when zoomed
    • the noscript element is present on all pages on the service, this may affect users who are trying to view alternate content on the site
  10. How we tested this service

    11. This service was last tested on 19 March 2021. The test was carried out by Prolifics Testing using both manual and automated test techniques to assess compliance against the desired AA rating against the WCAG 2.1 standard for accessibility. Manual checks included keyboard navigation, page zoom, contrast and high contrast / reverse mode, to simulate users relying on assistive technologies to access the pages within scope. Tests were carried out on Windows and Mac machines. Tools that were used include

    1. Narrator - Screen Reader (Windows)
    2. Apple Voiceover (Mac)
    3. Zoomtext
    4. Windows setting to invert colours using high contrast
    5. WebAim Wave
  11. What we’re doing to improve accessibility

    12. We’re fixing the problems that are stopping us from being compliant with the Web Content Accessibility Guidelines version 2.1A standard. We intend to have this done by January 2022.

    We’ll then move on to fixing the problems that are stopping us being compliant with the Web Content Accessibility Guidelines version 2.1 AA standard.
